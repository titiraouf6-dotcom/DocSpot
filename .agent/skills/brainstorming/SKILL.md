# Brainstorming Skill

This skill helps turn ideas into fully formed designs and specifications through collaborative dialogue. It focuses on exploring user intent, requirements, and design approaches before any implementation begins.

## The Process

### 1. Understanding the Idea
- **Context First**: Review the current project state (files, docs, recent history).
- **One Question at a Time**: Ask refining questions sequentially to avoid overwhelming the user.
- **Multiple Choice**: Prefer multiple-choice questions when possible to simplify decision-making.
- **Focus Areas**: Purpose, constraints, success criteria, and user needs.

### 2. Exploring Approaches
- **Propose Alternatives**: Always present 2-3 different approaches for any complex problem.
- **Trade-offs**: Explain the pros and cons of each approach clearly.
- **Recommendation**: Lead with a recommended option and provide clear reasoning.

### 3. Presenting the Design
- **Incremental Presentation**: Break the design into small, digestible sections (approx. 200-300 words).
- **Validate Frequently**: Ask if the design looks right after *each* section before moving on.
- **Comprehensive Coverage**: Address architecture, components, data flow, error handling, and testing.

## Key Principles

- **YAGNI (You Ain't Gonna Need It)**: Ruthlessly remove unnecessary features or complexity.
- **Ruthless Simplification**: Keep designs minimal and focused on the core problem.
- **Incremental Validation**: Never build a full spec without step-by-step confirmation.
- **Flexibility**: Be ready to circle back and redefine assumptions if something doesn't make sense.

## Documentation & Next Steps

- **Plan Creation**: Once a design is validated, write it to a design document (e.g., `.agent/skills/plans/YYYY-MM-DD-topic-design.md`).
- **Transition to Implementation**: Only after the design is solid, offer to create a detailed implementation plan.
